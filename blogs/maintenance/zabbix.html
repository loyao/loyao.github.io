    # 一．名词介绍

    监控工具 1.数据采集工具   数据采集 数据展示 数据分析 2.报警工具 主机：要监控的网络设备 主机组：主机的组，包含模板 监控项：监控指标 触发器（trigger）：判断监控项是否正常 事件（event）：值得关注的事情，比如触发器状态改变 动作（action）：对事件的处理方式 报警升级（escalation）:发送报警或者执行远程命令的方法 媒介（media）：发送详细的方式 通知（notification）：通过选定的媒介向用户发送的有关事件信息 前端：zabbix的web接口

    服务端跟被监控的对象进行通信， 对安装了agent进行监控

    # jvm监控

    ​ zabbix-java-gateway工具进行监控。可以去

    ​ 修改zabbix-java-gateway的配置文件，vim /etc/zabbix/zabbix_java_gateway.conf,修改 START_POLLERS，该配置为启动几个进程。修改zabbix-server的配置文件，vim /etc/zabbix/zabbix_server.conf，在JavaGateway那一行，将安装了zabbix_java_gateway的机器的ip地址填写，StartJavaPollers修改为和前面的配置启动进程数一样就可以。

    ​ 开启tomcat的jmx，首先关闭tomcat,进入tomcat的 bin目录 ，修改 修改catalina.sh ， 找到“#—–Execute The Requested Command” ，添加如下的配置，然后重新启动tomcat. tomcat8.5之后需要写在一行才能生效！！！！！。

    CATALINA_OPTS="$CATALINA_OPTS -Dcom.sun.management.jmxremote -Djava.rmi.server.hostname=192.168.1.246 -Dcom.sun.management.jmxremote.port=12345 -Dcom.sun.management.jmxremote.ssl=false -Dcom.sun.management.jmxremote.authenticate=false"

ip是你要监控的tomcat所在服务器的ip地址
端口号，是你要开启的监控端口号,zabbix默认是12345。
ssl，false表示不使用ssl链接
authenticate，false表示不使用监控，即不需要用户名和密码

    # 性能优化

    对变化不大的监控项，可以增加监控的时间间隔。

    配置文件中的MwareCacheSize 缓存空间，提高访问性能。

    # 三．SNMP监控

    snmp是简单网络管理协议（SNMP） 是专门设计用于在 IP 网络管理网络节点（服务器、工作站、路由器、交换机及HUBS等）的一种标准协议，它是一种应用层协议。

    下载地址：https://sourceforge.net/projects/net-snmp/files/net-snmp/

    安装选择源码安装，在进入了解压的目录之后，./configure执行安装相关，会选择相关信息，

    //选择snmp协议版本，默认第3版 Defaultversion of SNMP to use (3):3

    //系统联系信息 System Contact Information (@@no.where)

    //系统位置 System Location (Unknown) //日志位置 Location to write logfile (/var/log/snmpd.log) //持久日志存贮位置 Location to write persistent information

    然后进行编译和安装

    Yum 安装
    安装net-snmp net-snmp-utils
    开启配置
    sed -i.ori '57a view systemview included .1' /etc/snmp/snmpd.conf
